Receive Flow

三步完成握手

1

初始化接收端

系统生成 6 位 PIN，并开始广播设备信息与监听 QUIC 端口。

2

等待发送端发现

另一台设备输入 PIN 后会自动扫描与配对，雷达动画表示当前处于待命阶段。

3

持续接收队列

接收端支持队列里的后续文件，传输完成后历史会自动写入本地。

Settings / Runtime Configuration

把服务参数变成可视化配置

设备名、端口、并发流数和分块大小都会写入本地配置文件。除了 Web 监听地址需要重启外，其余设置会对后续新任务即时生效。

Apply Rules

设置生效说明

1

即时生效

设备名称、QUIC 端口、并发流数、块大小会应用到后续新发起的任务。

2

需要重启

Web 主机地址与 Web 端口会保存到本地，但当前进程需要重启后才切换监听地址。
